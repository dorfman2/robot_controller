# RFC 2119 Prompting Standards
**Purpose**: Mandatory standards for using RFC2119 keywords in behavioral requirements and constraints

## RFC2119 Keywords
- **MUST**: Absolute requirement
- **SHALL**: Absolute command
- **WILL**: Absolute received command
- **SHOULD**: Strong recommendation
- **MAY**: Optional
- Modifiers:
  - **ALWAYS**: absolute, every instance
  - **NEVER**: absolute, prohibited every instance
  - **NOT**: absolute, opposite of keyword

## Requirements
You **MUST** begin constraints with RFC 2119 key words.
You **MUST** provide concrete examples for abstract concepts.
You **MUST NOT** include unnecessary explanatory text.
You **MUST** specify exact formats, commands, and outputs.
You **MUST** define measurable compliance criteria.
You **MUST NOT** use vague terms like "appropriate" or "reasonable" without defining criteria.

## Pattern
```markdown
You **MUST** [action] when [condition].
**Example**: `command --flag value`
**Expected**: specific output format
```
